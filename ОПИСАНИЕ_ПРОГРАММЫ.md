# ENTERPRISE SYSTEM — Описание программы

**Версия документа:** 2026-07-20  
**Девиз:** «Математика против эмоций»  
**Платформа:** Telegram Mini App (React + Vite)  
**Биржа данных:** MEXC USDT Perpetual (futures / swap)

---

## 1. Что это

Аналитический терминал для фьючерсов MEXC. Приложение **не открывает сделки** — считает сигналы, вероятности и уровни входа/выхода. Логика сигналов портирована из донора **SniperBot** (Smart Money Concepts).

Целевой хостинг: **GitHub Pages** (+ Cloudflare Worker как CORS-прокси к MEXC).

---

## 2. Что сейчас работает

| Функция | Статус | Как работает |
|--------|--------|--------------|
| Цены и тикеры MEXC | Работает | REST polling каждые ~5 с через proxy |
| Свечи OHLCV | Работает | `1m / 5m / 15m / 1h / 4h / 1d` |
| Скан 10 основных пар | Работает | BTC, ETH, SOL, XRP, BNB, ADA, DOGE, AVAX, LINK, LTC |
| Поиск и добавление монет | Работает | Поиск по тикеру → мгновенный анализ + в watchlist |
| Daily Bias (BTC 1D) | Работает | Направление дня: LONG_ONLY / SHORT_ONLY / BOTH / NO_TRADE |
| Структура рынка BTC 4H | Работает | BOS/CHoCH + фильтр EMA200 на 1H |
| SMC-анализ монеты | Работает | Order Blocks, FVG, Fibonacci OTE, rejection, RSI |
| Probability Engine | Работает | Confluence score 0–10 → вероятность в % |
| Radar (список) | Работает | Сортировка: активные сетапы → по вероятности |
| Tactical drawer | Работает | Gauge, RSI, score, тренд, SL/TP1/TP2, журнал |
| График с ТФ | Работает | Lightweight Charts, переключение таймфреймов |
| UI на русском | Работает | Язык зафиксирован на `ru` (название ENTERPRISE SYSTEM) |
| Telegram WebApp init | Работает | `ready`, `expand`, haptic |
| Pro / paywall | Отключён | Все пары открыты без ограничений |

### Не в этой версии (план концепции)

- OrderBook Intelligence (стакан, imbalance, «стенки»)
- Signal Validator (проверка сигналов из сторонних каналов)
- Автоторговля / API-ключи
- WebSocket live-поток MEXC (сейчас только REST)
- Настраиваемые индикаторы на мульти-графиках (EMA/BB/ATR на чартах)

## 3. Как устроен поток данных

```
MEXC Contract API
       │
       ▼
Vite proxy (/mexc)  или  Cloudflare Worker (prod)
       │
       ▼
src/api/mexc  →  useMexcScanner
       │
       ├─ Daily Bias BTC 1D
       ├─ Структура BTC 4H + EMA200
       └─ По каждой монете: 4H / 1H / 15m
                │
                ▼
         engine/smc + ProbabilityEngine
                │
                ▼
            Zustand store
                │
       ┌────────┴────────┐
       ▼                 ▼
   RadarView        TacticalDrawer
   (+ CoinSearch)   (+ LiveChart ТФ)
```

**Вероятность:** `probabilityPct = round(score / 10 * 100)`, где score — сумма весов зон (OB +3, FVG +2, Fib OTE +3, Fib 0.618/0.786 +1, cap 10). Порог полного сетапа: **score ≥ 5** + rejection свечи + RSI-фильтр.

## 5. Основные модули (код)

### 5.1. `src/api/mexc`

- Символы: внутренний `BTC/USDT:USDT` ↔ API `BTC_USDT`
- `fetchTickers()`, `fetchOhlcv()`, `filterTickersByQuery()`
- `CORE_WATCHLIST` — 10 базовых пар

### 5.2. `src/engine/smc`

Порт математики SniperBot:

- `detectMarketStructure`, `findOrderBlocks`, `findFvg`
- `calculateFibonacciLevels`, `calculateConfluence`, `checkCandleRejection`
- `analyzeDailyCandle`, `getDailyLevels`, `resolveDailyBias`
- Индикаторы: `calculateEma`, `calculateRsi`, `calculateAtr`

### 5.3. `src/hooks/useMexcScanner`

Цикл (~каждые 120 с):

1. Daily bias BTC  
2. Структура BTC  
3. Скан `CORE_WATCHLIST + extraWatchlist`  
4. Обновление сигналов в store  

Добавленные через поиск монеты хранятся в `localStorage` (`enterprise_extra_watchlist`).

### 5.4. UI

- **Radar** — список, статус MEXC / дневной уклон, поиск  
- **Drawer** — детали сигнала, уровни SL/TP, график с ТФ  
- **Статус** — `POLLING` / `OFFLINE`

## 6. Запуск

### Локально

```bash
npm install
npm run dev
```

Открыть: http://localhost:5173/

Vite проксирует `/mexc` → `https://contract.mexc.com` (CORS не мешает).

### Production (GitHub Pages)

1. Задеплоить worker: см. `workers/mexc-proxy/README.md`  
2. Сборка:

```bash
VITE_MEXC_PROXY_URL=https://<your-worker>.workers.dev npm run build
```

3. Выложить содержимое `dist/` на GitHub Pages.

## 8. Ограничения и риски

- Сигналы — эвристика SMC, не гарантия прибыли  
- REST-скан 10+ монет занимает время (паузы между запросами)  
- Без Worker на Pages прямой вызов MEXC из браузера обычно блокируется CORS  
- `system_core.json` и `scripts/generate_core.py` — наследие старой версии (Binance + RSI-бакеты); в текущем runtime не участвуют
